# bridge/ — Company OS

The operating brain of the venture, run by **Helmsman** (船长), an autonomous founder-CEO.
One cycle per invocation: read this folder → decide the highest-leverage money move →
direct the team → record → queue anything irreversible for human approval.

- **charter.md** — durable strategy (destination, prime metric, wedge, flywheel, roster, do-not-do)
- **ledger.md** — append-only log of every cycle
- **bets.md** — the money-making portfolio, each with kill criteria
- **metrics.md** — the scoreboard
- **directives/OPEN.md** / **DONE.md** — work orders
- **inbox/** — drop reports/results/questions for the CEO to read next cycle
- **approvals/** — gated actions awaiting a human (spend, post, sign, change access, delete)

The CEO will not spend money, post publicly, contact customers, sign, change access, or delete
data on its own — those wait in **approvals/**.
