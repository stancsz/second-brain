# G09 — WIP chain: bi-temporal validity + `--as-of` recall (R11)

## Goal
Close R11: `--as-of <date>` recall returns the historically-valid state;
superseded facts are excluded. Match Zep/Graphiti's bi-temporal bar (per-fact
validity windows; a contradiction CLOSES the old window rather than deleting it;
point-in-time queries) at the core level, with our simpler single-graph model.

## Architecture decision (why a derived table, not columns on `concepts`)
The OKF fields `sb_valid_from` / `sb_valid_to` / `sb_supersedes` already
serialize through okf.py and carry through `concepts.metadata` (bundle
`_SB_FIELDS`). Following the established G08 (subjects) / G10 (affect) pattern,
the SQLite side is a **derived index table** (`validity`) rebuilt from
`concepts.metadata` by `rebuild_validity_index()`, NOT new columns on the
`concepts` table. This keeps the core `concepts` shape stable (zero risk to the
schema-rename / bundle-rebuild / okf-spec verifiers) and keeps "metadata is
truth, derived tables are rebuildable indexes" invariant.

`validity(concept_id PK, valid_from TEXT, valid_to TEXT, supersedes TEXT,
FK→concepts ON DELETE CASCADE)`. ISO date/datetime strings (lexicographically
comparable). A Concept with no validity row is "valid from its `created_at`,
still valid" (valid_to = NULL).

## Milestones

### M1 — bi-temporal storage + supersession write-path  (THIS ITER)
- `validity` table (schema.sql) + `ix_validity_window`.
- `add(..., sb_valid_from, sb_valid_to, sb_supersedes)` → metadata + sync row.
- `update(..., sb_valid_from=_UNSET, sb_valid_to=_UNSET, sb_supersedes=_UNSET)`.
- `validity(id)` getter; `_sync_validity_for`; `rebuild_validity_index()`
  (wired into `bundle.rebuild`).
- `supersede(old_id, title, content, ..., as_of=None)` convenience write-path:
  creates a NEW Concept with `sb_supersedes=old_id` and `sb_valid_from=as_of`,
  and sets the OLD Concept's `sb_valid_to=as_of` (default as_of = today). This
  is the "contradiction closes the old window" operation.
- Verifier `temporal-validity` (M1): validity persists + round-trips through
  bundle export/rebuild; `supersede()` closes the old window and links the new;
  FK cascade; partial windows (valid_from only) persist.
- Does NOT yet satisfy R11 (no `--as-of` query) — that is M2.

### M2 — `--as-of` recall query + CLI  (NEXT ITER, satisfies R11)
- `recall_as_of(as_of, query=None, collection=None, limit=...)`: returns live
  Concepts whose validity window contains `as_of`
  (`COALESCE(valid_from, created_at) <= as_of AND (valid_to IS NULL OR
  valid_to > as_of)`), optionally AND an FTS match. Superseded/expired excluded.
- `brain recall --as-of <date> [query]` CLI subcommand.
- Verifier `temporal-asof` (the R11 binding verifier): insert a superseded
  history (lived-in-NYC → lives-in-SF via `supersede`), assert as-of in the NYC
  window returns NYC not SF, as-of in the SF window returns SF not NYC, boundary
  (valid_to exclusive) handled, timeless notes returned for as-of ≥ creation.
- Mark R11 DONE in RELEASE.md.
